# 🔧 Ansible - Questions 1

## Questions

### 1. 🎯 What is Ansible? 🟢

A) 🔧 Configuration automation tool

B) 🔄 Monitoring system

C) 📦 Package manager

D) 🌐 Web server

**Correct Answer**: A) 🔧 Configuration automation tool

> 💡 Ansible automates configuration, deployment, and application management. Like having an assistant that configures all your servers!

### 2. 📝 What format does Ansible use for configuration files? 🟢

A) 📝 YAML

B) 📄 JSON

C) ⚙️ XML

D) 🔧 INI

**Correct Answer**: A) 📝 YAML

> 📘 YAML is human-readable and easy to write. Like writing simple but powerful notes!

### 3. 🎭 What is a playbook in Ansible? 🟢

A) 🎭 File that defines tasks to execute

B) 🔧 Configuration script

C) 📦 Installation package

D) 🌐 Network file

**Correct Answer**: A) 🎭 File that defines tasks to execute

> ⚡ Playbooks are like scripts that Ansible follows to configure servers. Like a conductor for your infrastructure orchestra!

### 4. 🎯 What is an inventory in Ansible? 🟡

A) 📋 List of hosts to manage

B) 🔧 Configuration file

C) 📦 Server database

D) 🌐 Control server

**Correct Answer**: A) 📋 List of hosts to manage

> 💡 The inventory defines which servers Ansible should manage. Like a guest list for your configuration party!

### 5. 🔧 What is a module in Ansible? 🟡

A) 🔧 Specific functionality unit

B) 📦 Software package

C) ⚙️ Configuration file

D) 🌐 Network service

**Correct Answer**: A) 🔧 Specific functionality unit

> 📘 Modules are the tools Ansible uses to perform specific tasks. Like having a specialized toolbox!

### 6. 🔄 What does idempotency mean in Ansible? 🟡

A) 🔄 Run multiple times without changing result

B) 🔧 Configure once

C) 📦 Install packages

D) 🌐 Connect servers

**Correct Answer**: A) 🔄 Run multiple times without changing result

> 🎯 Ansible is idempotent - you can run the same playbook many times without issues. Like a button that always does the right thing!

### 7. 🌐 What is a group in Ansible? 🟢

A) 🌐 Logical collection of hosts

B) 🔧 Main server

C) 📦 Configuration package

D) ⚙️ Network file

**Correct Answer**: A) 🌐 Logical collection of hosts

> 💡 Groups allow organizing hosts by function or location. Like organizing your contacts by categories!

### 8. 🔐 What is Ansible Vault? 🟡

A) 🔐 Encryption system for sensitive data

B) 🔒 Network firewall

C) 🛡️ Antivirus

D) 🔑 Password manager

**Correct Answer**: A) 🔐 Encryption system for sensitive data

> ⚡ Vault encrypts passwords and sensitive data in your playbooks. Like a safe for your secrets!

### 9. 📊 What is a handler in Ansible? 🟡

A) 📊 Task that runs only when notified

B) 🔧 Configuration script

C) 📦 Software package

D) 🌐 Network service

**Correct Answer**: A) 📊 Task that runs only when notified

> 📘 Handlers run only when something changes. Like a service that only comes when you call it!

### 10. 🔄 What is a role in Ansible? 🟡

A) 🔄 Reusable task collection

B) 🔧 Installation script

C) 📦 Configuration package

D) 🌐 Network service

**Correct Answer**: A) 🔄 Reusable task collection

> 💡 Roles organize playbooks into reusable components. Like having LEGO pieces to build your infrastructure!

### 11. 🌐 What is a fact in Ansible? 🟢

A) 🌐 Host system information

B) 🔧 Configuration data

C) 📦 Environment variable

D) ⚙️ Network parameter

**Correct Answer**: A) 🌐 Host system information

> 🎯 Facts are automatic data about the system (IP, OS, etc.). Like having a detective that automatically gathers information!

### 12. 🔧 What command runs a playbook? 🟢

A) 🔧 `ansible-playbook`

B) 📦 `ansible-run`

C) ⚙️ `ansible-execute`

D) 🌐 `ansible-deploy`

**Correct Answer**: A) 🔧 `ansible-playbook`

> ⚡ `ansible-playbook` executes Ansible playbooks. Like the command that starts all the automation!

### 13. 🔐 What is become in Ansible? 🟡

A) 🔐 Elevate user privileges

B) 🔒 Change user

C) 🛡️ Activate security

D) 🔑 Manage permissions

**Correct Answer**: A) 🔐 Elevate user privileges

> 📘 `become` allows running tasks with elevated privileges (like sudo). Like asking for special permissions when you need them!

### 14. 📊 What is a template in Ansible? 🟡

A) 📊 File with dynamic variables

B) 🔧 Configuration model

C) 📦 Installation package

D) 🌐 Network file

**Correct Answer**: A) 📊 File with dynamic variables

> 💡 Templates generate files with dynamic content based on variables. Like having a mold that adapts to each situation!

### 15. 🔄 What is a loop in Ansible? 🟢

A) 🔄 Repeat a task multiple times

B) 🔧 Infinite loop

C) 📦 Installation cycle

D) 🌐 Network connection

**Correct Answer**: A) 🔄 Repeat a task multiple times

> 🎯 Loops allow running the same task with different values. Like doing the same thing many times without repeating code!

### 16. 🌐 What is a variable in Ansible? 🟢

A) 🌐 Value that can change

B) 🔧 Fixed configuration

C) 📦 Network parameter

D) ⚙️ System data

**Correct Answer**: A) 🌐 Value that can change

> ⚡ Variables allow customizing the behavior of your playbooks. Like having settings that change according to the environment!

### 17. 🔐 What is a condition in Ansible? 🟡

A) 🔐 Run tasks only if condition is met

B) 🔒 Verify security

C) 🛡️ Validate configuration

D) 🔑 Check permissions

**Correct Answer**: A) 🔐 Run tasks only if condition is met

> 📘 Conditions allow running tasks only when necessary. Like having intelligent logic in your automation!

### 18. 📊 What is a tag in Ansible? 🟢

A) 📊 Label to organize tasks

B) 🔧 Configuration mark

C) 📦 Package identifier

D) 🌐 Network label

**Correct Answer**: A) 📊 Label to organize tasks

> 💡 Tags allow running only certain parts of a playbook. Like having tabs to organize your work!

### 19. 🔄 What is delegate_to in Ansible? 🟡

A) 🔄 Run task on specific host

B) 🔧 Delegate configuration

C) 📦 Transfer package

D) 🌐 Redirect connection

**Correct Answer**: A) 🔄 Run task on specific host

> 🎯 `delegate_to` runs a task on a different host than the current one. Like asking someone else to do the work!

### 20. 🌐 What is gather_facts in Ansible? 🟢

A) 🌐 Collect system information

B) 🔧 Gather configuration

C) 📦 Get packages

D) ⚙️ Manage data

**Correct Answer**: A) 🌐 Collect system information

> ⚡ `gather_facts` collects automatic information about hosts. Like taking inventory of what you have!

### 21. 🔐 What is vault_password_file in Ansible? 🟡

A) 🔐 File containing Vault password

B) 🔒 Security configuration file

C) 🛡️ Permissions file

D) 🔑 Keys file

**Correct Answer**: A) 🔐 File containing Vault password

> 📘 Allows specifying where the password is to decrypt Vault files. Like having your safe key in a secure place!
